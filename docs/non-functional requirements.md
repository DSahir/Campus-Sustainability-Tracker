# Non-Functional Requirements

## Performance
Support at least 500 concurrent users. Dashboards should load within 2 seconds, and sustainability metrics refresh within 5 seconds after updates.  

## Security
All communication uses HTTPS and sensitive data is encrypted in transit and at rest. Access is controlled via role-based authentication (RBAC).  

## Availability
Maintain at least 99% uptime. The system should recover automatically in case of failure.  

## Scalability
Monitor up to 100 buildings across 5 campuses and process 1,000 sensor updates per minute without performance degradation.  

## Usability
Provide a responsive interface for desktop with clear dashboards and accessible design.
